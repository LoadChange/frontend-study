### 创建对象有几种方法
```
1.  // 第一种方式：字面量
    var o1 = {name: 'o1'};
    var o2 = new Object({name: 'o2'});

2.  // 第二种方式：构造函数
    var M = function (name) { this.name = name; };
    var o3 = new M('o3');

    任何函数只要被new使用了，这个函数就是一个构造函数

3.  // 第三种方式：Object.create
    var p = {name: 'p'};
    var o4 = Object.create(p);

```

> 构造函数使用new来生成一个实例，任何函数都可以当成构造函数，构造函数也是函数，函数都有一个prototype属性，这个是在声明一个函数的时候，js引擎自动加上去的一个属性，prototype指向的就是原型对象。

> 原型对象中有一个构造器constructor，构造器默认是声明的函数

> 实例对象中有一个__proto__,指向的是原型对象

> 原型链：从一个实例对象开始往上找构造这个对象，相关联的对象，依次向上查找，以此类推一直到object.prototype终止，object.prototype是原型链的顶端。（使用prototype、__proto__属性完成查找）

> 原型链和原型对象的作用：在构造函数中增加属性，在实例对象中就可以共用这些属性，假如实例对象有共用的方法，每个实例对象都拷贝一份这样的方法，这样太占内存，没有必要，就可以添加在原型链上。

```
通过原型链的方式，找到原型对象，原型对象上的方法是被不同实例所共有的。

在访问一个实例的时候，在使用一个方法，在该实例上没有找到这个方法，它会往原型对象上查找__proto__,直至Object.prototype。如果一直没找到，原路返回，报告没有找到该方法或属性，如果在中间任意环节找到了，停止向上查找，直接返回。
```
**构造函数（函数）才有prototype,对象是没有的。只有实例对象有__proto__(函数也是对象，函数的__proto__是Function.prototype,函数也就是Function的实例对象)**
